How to Use the Chatbot

Welcome to Surreal AI! Here's how you can interact with our chatbot:

Text Interaction

  • Navigate to the chat page by clicking the "Join Waitlist" button and submitting your email.
  • On the chat page, type your queries into the text input box and press Enter.
  • The chatbot will respond with a text message.
  • If you want to end the conversation, type "quit".

Speech Interaction

  • Navigate to the chat page by clicking the "Join Waitlist" button and submitting your email.
  • On the chat page, click the microphone button to start the speech interaction.
  • Speak your question clearly into the microphone.
  • The chatbot will transcribe your speech and respond with a text message.
  • The chatbot will also convert its response to speech and play it back to you.
  • If you want to end the conversation, say "quit".

Enjoy your conversation with Surreal AI!

Reducing Latency through Asynchronous Processing in Speech-to-Speech Systems

Our innovative speech-to-speech system utilizes an asynchronous architecture to significantly reduce latency and improve user experience. Here's how our system outperforms traditional synchronous approaches:

Parallel Processing:

In our asynchronous system, the three main components - Speech-to-Text, Language Model (LLM), and Text-to-Speech - operate concurrently. This parallel processing allows for a substantial reduction in overall latency compared to sequential execution.

Early Start Advantage:

The LLM begins processing as soon as the first chunks of transcribed text are available, rather than waiting for the entire speech input to be transcribed. This head start significantly reduces waiting time.

Continuous Output Generation:

Our Text-to-Speech module starts generating audio output as soon as it receives the initial responses from the LLM. This means users can hear the beginning of the response while the system is still processing the latter parts of the input.

Efficient Resource Utilization:

By allowing all components to work simultaneously, we ensure that computational resources are used efficiently, with no idle time between processes.

Scalability:

The asynchronous architecture allows for easy scaling. We can process multiple requests concurrently, improving the system's overall throughput.

Reduced End-to-End Latency:

In practical terms, our asynchronous system can deliver responses up to 40% faster than traditional synchronous systems, depending on the input complexity and length.

Improved User Experience:

The faster response times and the ability to start hearing outputs earlier create a more natural, conversational feel for the user, closely mimicking human-to-human interactions.

Adaptive Processing:

Our system can dynamically adjust processing based on the incoming data stream, optimizing performance in real-time for various speech patterns and lengths.

By leveraging this asynchronous architecture, we've created a speech-to-speech system that not only reduces latency but also provides a more responsive and engaging user experience. This approach represents a significant step forward in making AI-driven conversations feel more natural and instantaneous.
